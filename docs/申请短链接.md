# 申请短链接指南

本指南将帮助您了解如何在本项目中添加和管理短链接。

## 短链接添加方式

### 第一步：使用 SHA512 生成唯一识别码

为了确保识别码的唯一性，推荐使用 SHA512 哈希算法生成：

1. **选择目标URL**
2. **计算哈希值**：对目标URL进行 SHA512 哈希计算
3. **提取前缀**：取哈希值的前5-8位作为识别码
4. **添加到配置文件**：按照方法二的格式添加到 `redirects.json`

### 第二步：修改 JSON 文件

1. **定位配置文件**：打开项目根目录下的 `data/redirects.json` 文件
2. **添加新条目**：在 `redirects` 对象中添加新的短链接配置
3. **格式要求**：
   - 键名：5-8位的唯一识别码
   - 值：包含 `target`（Base64加密的目标URL）和 `created`（创建时间）的对象

## Base64 加密方法

目标URL需要使用 Base64 加密后存储：

### 在线加密工具

- [Base64 编码/解码](https://www.base64encode.org/)
- [URL Base64 编码](https://www.urlencoder.io/base64/)

### 命令行加密（使用 Node.js）

```bash
node -e "console.log(Buffer.from('https://example.com').toString('base64'))"
```

## 配置文件示例

```json
{
  "redirects": {
    "abc123": {
      "target": "aHR0cHM6Ly9leGFtcGxlLmNvbQ==",
      "created": "2026-04-18T12:00:00Z"
    },
    "def456": {
      "target": "aHR0cHM6Ly9nb29nbGUuY29t",
      "created": "2026-04-18T12:30:00Z"
    }
  }
}
```

## 注意事项

1. **识别码唯一性**：5-8位识别码必须唯一，不能重复
2. **URL格式**：目标URL必须包含 `http://` 或 `https://` 协议
3. **Base64格式**：确保使用标准的 Base64 编码
4. **时间格式**：创建时间使用 ISO 8601 格式（如 `2026-04-18T12:00:00Z`）
5. **安全性**：不要在配置文件中存储敏感信息

## 测试短链接

添加后，您可以通过以下格式访问短链接：

```
http://your-domain.com/u/?id={识别码}
```

例如：

```
http://your-domain.com/u/?id=abc123
```

## 错误处理

如果遇到以下情况，系统会显示相应的错误页面：

- **404错误**：短链接不存在
- **解码错误**：Base64解码失败
- **URL错误**：目标URL格式不正确
- **配置错误**：配置文件加载失败

## 最佳实践

1. **使用描述性识别码**：虽然可以使用随机字符，但建议使用有意义的识别码
2. **定期备份**：定期备份 `redirects.json` 文件
3. **版本控制**：将配置文件纳入版本控制系统
4. **测试验证**：添加新链接后，务必测试确认可以正常跳转

## 常见问题

### Q: 短链接不生效怎么办？

A: 检查识别码是否正确，Base64编码是否正确，目标URL是否包含协议

### Q: 如何批量添加短链接？

A: 可以编写脚本批量生成配置条目，然后更新 `redirects.json` 文件

### Q: 短链接可以统计访问次数吗？

A: 本项目为纯静态实现，不包含访问统计功能。如需统计，可考虑集成第三方分析工具，或者将域名托管到CloudFlare
